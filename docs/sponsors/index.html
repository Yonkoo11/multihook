Sponsor depth audit — honestly scored

Every integration claim has a file:line reference and a depth tier on a public 0-5 scale. We score ourselves the same way an outside auditor would. Tracks where we couldn't honestly hit depth 3 are marked DROPPED rather than name-dropped.

Depth scale: 0 = not used · 1 = name-drop only · 2 = wallet/auth-level use · 3 = one non-trivial SDK/API call (the floor for an honest claim) · 4 = two+ surface areas integrated, demoable · 5 = load-bearing, remove it and the demo breaks.
